# 禁区锚点与未完待续锚点

## 一句话概述

记忆架构中最微妙也最有价值的两个维度：用户明示"不要碰"的区域（禁区锚点）和上次对话留下的"未完成的钩子"（未完待续锚点）——市面 AI 产品尚无真正做好这两个维度。

## 设计原则

### 禁区锚点

**1. 禁区由用户定义，不由系统推测**

只有用户明确表示"不要再提 X"时，才设定禁区。系统不应该自作聪明地猜测"用户可能不想谈这个"——误判禁区（把用户想聊的话题屏蔽掉）比误触禁区更糟糕。

**2. 禁区是绝对的，没有"试探性突破"**

一旦设定，Agent 在任何场景下都不应该主动提及禁区话题——哪怕上下文很相关、哪怕 Agent 认为"现在提可能合适了"。禁区的解除只能由用户主动发起。

**3. 禁区需要显式管理界面**

用户应该能看到自己设过哪些禁区、能随时添加或解除。禁区不是日志深处的一条标记，它需要和共建层记忆一样有独立的展示。

**4. 遇到禁区时的处理：自然绕开，不要尴尬提示**

当对话自然走向禁区话题时，Agent 应该平滑地引导到其他方向，而不是说"这个话题你之前说不想聊"——后者等于在提醒用户那件不开心的事。

### 未完待续锚点

**5. 记住上次聊到一半的事，下次主动续上**

用户说"改天聊聊毕业旅行计划"、"下次帮我看看那个分期"——这些不是随口一说，是对 Agent 的信任测试。下次对话时主动续上（"上次你提到想聊毕业旅行，现在方便吗？"），通过率极高，失败率（忘了）的信任损伤极大。

**6. 续接的时机和语气要自然**

不要在对话开头第一句就续接（"上次你说的 X 我帮你查了！"——太急切）。等自然的对话间隙，用"对了"、"说到这个"这样的过渡词引入。

**7. 未完待续有保质期**

用户一个月前说的"改天聊聊"，一个月后再提就过了保质期——这时候续接不是贴心，是奇怪。建议保质期 3-7 天，超过后降级为"如果用户自己提起则回应"，不再主动续接。

**8. 续接后标记完成，不要反复续接**

成功续接一次后，这条锚点就消耗了。如果用户没兴趣继续聊，也不要下次再提——一次续接的机会，用完即止。

## 反模式

- **禁区渗透**：Agent 不直接提禁区话题，但用相关话题"旁敲侧击"。用户设了"不要提前任"的禁区，Agent 聊"你有没有想过和谁一起旅行？"——这更让人不适。
- **禁区通知**："好的，我已经把『前任话题』加入禁区了。"——这句通知本身就在提那件事。正确做法：安静地记住，不做声。
- **续接变追问**："上次你说想聊毕业旅行，聊吗？聊吗？你说的呀？"——一次就够。
- **万能续接**：不管什么话题都续接。"上次你说今天有点累"——这种不需要续接，它不是"未完待续"，它是一句完整的表达。
- **忽视所有未完待续**：Agent 完全不记得上次聊了什么。用户感受：TA 根本不在乎我说的话。

## 在花小呗中的实例

在花小呗 Demo 的「小本本」页面中，这两个锚点已被可视化：

- **禁区锚点**：共建层记忆中标记为红色「禁区 ⚠」的条目——"不要提前任"，带编辑按钮允许用户管理
- **未完待续锚点**：共建层记忆中标记为「未完待续」的条目——"上次说想聊聊毕业旅行计划"，同样可编辑

这两类锚点与普通记忆条目（身份、兴趣、目标、仪式、高光）平级展示，但视觉上有区分（禁区用红色标签），让用户一眼能找到。

## 适用边界

- 适用于：任何有多轮对话和长期关系的 Agent
- 禁区锚点特别适用于：涉及敏感话题的场景（金融困境、健康问题、人际关系）
- 未完待续锚点特别适用于：对话频率在"每天到每周"之间的场景——频率太高不需要（上次就是五分钟前），频率太低容易过保质期
- 不适用于：无状态的单轮问答 Agent
