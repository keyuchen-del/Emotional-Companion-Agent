# 花小呗 · 花呗特有设计决策

本文档记录与花呗品牌/产品深度耦合的设计决策，这些决策不适合抽象为通用原则，但对理解本 case 的落地逻辑至关重要。

## 吉祥物形态

花小呗采用 Q 版花朵形态（🌸），设计考量：

- **为什么是花朵**：「花」呗的品牌联想，用户零认知成本
- **渐变色方案**：`radial-gradient(#FFF0F2 → #FFD6DD → #FFBCC8)`，温暖但不幼稚
- **绿色小脚**：`#B8DFAC`，打破纯粉色的"少女感"，增加一点生机和中性化
- **两个尺寸**：Header 区 44px 小头像 + 对话区 110px 大形象，大形象配脚部动画增加"在场感"

## UI 设计语言对齐

严格对齐支付宝 / 花呗现有产品 UI：

- **主色**：`#FF4D4F`（花呗红），用于亲密度分数、CTA 按钮、进度条、Tab 激活态
- **气泡**：Bot 白色圆角（4px 18px 18px 18px），User 淡紫色圆角（18px 18px 4px 18px）
- **字体栈**：`-apple-system, "PingFang SC", "SF Pro Text"`，严格跟随支付宝
- **底部结构**：功能入口芯片 + 输入框 + 麦克风 + AI 生成声明 + Tab 栏，复刻支付宝智能助手布局
- **Header Pill**：白色胶囊体含吉祥物头像 + 亲密度分数 + 用户头像，毛玻璃效果 `backdrop-filter: blur(12px)`

## 拒额场景话术

五幕剧在花呗场景中的具体话术设计：

| 幕 | 话术要点 |
|---|---|
| 通知 | 「呜...小A，这次我没帮你争取到临时提额...」——用"我没帮你争取到"而非"系统拒绝了"，让 Agent 站在用户一边 |
| 共情 | 「花了这么多时间准备，被卡在这一步，换我我也会『诶』一下。」——「诶」比「难过」更口语、更真实 |
| 解释 | 「这不是你不够好，是时机不对。」——这句是整个话术中最关键的一句，把"被评判感"转化为"时机问题" |
| 行动 | 三选项：备用金（利息更低）/ 分期（首期 X 元）/ 其他支付（家人代付）——备用金排第一因为它对用户最有利，符合三序原则 |
| 闭环 | 「演唱会结束后某天，我想听你说现场怎么样」——留一个"期待你回来"的钩子，不是留一个"下次再申请"的商业钩子 |

## 花呗特有功能入口

底部芯片（Bottom Chips）对应花呗生态内的核心功能：

- **消费人格速鉴**：基于花呗消费数据的性格测试，轻量互动
- **快速涨额**：引导用户完成可提额行为
- **查额度**：花呗核心功能的快捷入口
- **涨亲密度**：引导用户了解亲密度机制

这些入口在通用框架中的对应物是"场景化快捷功能"，但具体内容完全由产品生态决定。

## 亲密度权益清单

权益与花呗 / 支付宝生态绑定：

| 权益 | 来源生态 |
|---|---|
| 闪购红包 20-10 | 花呗消费场景 |
| 闲鱼 1 元红包 | 支付宝生态（闲鱼） |
| 分期免息 3 期 | 花呗分期 |
| 分期专享额度券 | 花呗分期（不占原额度） |
| 打车 75 折 | 支付宝生态（滴滴） |

在通用框架中，这对应 [05 关系成长体系](../../docs/05-relationship-growth.md) 中的"权益作为关系成长的外化手段"。
