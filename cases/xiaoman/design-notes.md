# 小满 Mun · 品牌特有设计决策

本文档记录与「小满 Mun」品牌深度耦合的设计决策。这些决策不适合抽象为通用原则，但对理解本 case 的落地逻辑至关重要。

## 品牌定位

「小满」取自二十四节气，原义「麦粒渐满、尚未全熟」，引申为**小满即圆满，刚刚好的拥有**。

- **为什么是「小满」**：年轻人消费陪伴产品最大的伦理风险是「越陪越鼓励花钱」。用「小满」做品牌内核，把价值观钉死在**节制而非刺激消费**——陪你买得值、买得开心，也陪你忍住、退掉、攒下来。
- **与花小呗的区别**：花小呗深耦合花呗信贷生态（额度、提额、分期）；小满是中性品牌，**不碰借贷**，只做「消费决策 + 情绪陪伴」。

## 吉祥物形态

小满采用奶油团子形态（纯 CSS 绘制，非 emoji），设计考量：

- **为什么是团子 + 新芽**：团子 = 「满」的圆润饱满；顶上新芽 = 「渐满、在生长」的关系隐喻，呼应节气意象
- **配色方案**：体 `radial-gradient(#FFF6E2 → #FFE2A8 → #FFC76B)`（奶油到蜂蜜），新芽 `#5FBE63→#8AD583`，腮红 `rgba(255,120,150,.5)`
- **纯 CSS 绘制的原因**：吉祥物需要眨眼、拍拍挤压、随场景切换表情（sad/think/joy），emoji 无法做到。三模态情绪表达必须可编程
- **三种表情**：joy（开口笑+眯眼）/ think（单眼眯、嘴成短横）/ sad（倒八字眼+下弧嘴+流泪动画），随五幕剧情绪推进切换

## UI 设计语言

刻意区别于花小呗的金融红，建立年轻、暖、生活感的视觉系统：

- **主色**：蜂蜜橙 `#FF9F45`（CTA、满值、进度）；辅色 新芽绿 `#5FBE63`、莓粉 `#FF6F91`、天蓝 `#5FB0E8`
- **底色**：奶油 `#FCF7EC`，叠加多向径向渐变光晕 + SVG noise 颗粒，营造质感而非纯色平涂
- **字体栈**：Baloo 2（数字/满值，圆体有性格）+ Fredoka（拉丁/品牌名）+ PingFang SC（中文正文），刻意避开 Inter/Roboto 等 AI 通用脸
- **气泡**：Bot 白色圆角（6px 20px 20px 20px），User 蜂蜜橙渐变（20px 20px 6px 20px）
- **底部结构**：场景快捷芯片 + 输入框 + 发送 + AI 声明 + Tab 栏

## 强交互设计

「交互感强」是本次重做的核心诉求，落地为：

- **拍拍涨满值**：点吉祥物 → 弹性挤压动画 + 漂浮「+1」+ 满值递增，把「关系靠互动养成」做成可感知动作
- **对话流**：建议气泡 → 用户气泡 → 打字三点 → Bot 回复，节奏拟人
- **满值环**：进入「满值」页时 SVG 环带弹性曲线充填
- **五幕剧情绪同步**：每推进一幕，吉祥物表情同步切换，进度条可点跳转

## 球鞋发售场景话术

五幕剧在「限量联名球鞋没抢到」场景中的具体话术：

| 幕 | 话术要点 |
|---|---|
| 通知 | 「唉…没抢到 手慢了半分钟，库存就空了。」——先共担情绪，不甩锅给用户手速 |
| 共情 | 「这种『差一点点』最难受了，换我也会捶大腿。」——把失望正常化 |
| 解释 | 「是限量 + 黄牛脚本抢，不是你不够快。」——把「被评判感」转成「客观难度」 |
| 行动 | 三选项：蹲二级市场（设溢价上限拦你）/ 等补货（官方两周后再放）/ 同系列替代款——平等呈现，无引导 |
| 闭环 | 「不管买没买，你开心才重要……过两天我回来问你缓过来没。」——留情绪钩子而非商业钩子 |

## 消费场景功能入口

场景快捷芯片对应年轻人高频消费需求：

- **值不值得买**：发链接 → 比价 + 真实评价 + 心愿单排序，辅助判断而非替你决定
- **降价提醒**：盯心愿单，到价/补货主动喊
- **月度消费复盘**：温柔版账单，「不审判」是明确设计原则
- **心愿单**：欲望的「冷静区」，对应未完待续锚点

在通用框架中，这些对应「场景化快捷功能」，但具体内容由消费陪伴这一赛道决定。
