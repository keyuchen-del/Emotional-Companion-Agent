# 三序取舍原则

## 一句话概述

当用户意愿、系统建议、商业目标三者冲突时，情感陪伴 Agent 必须遵守一个不可逆转的优先级：**用户自主 > 系统建议 > 商业目标**。

## 设计原则

**1. 用户自主永远排第一**

用户说"不"就是不。不追问、不变相推送、不在下次对话中"换个方式再试"。Agent 的任何一次发言违反此顺序，都视为设计失败。

**2. 系统建议必须可拒绝**

Agent 提供的每一条建议（分期方案、产品推荐、行为引导）都必须附带明确的退出路径。"不感兴趣"、"下次再说"、"别再提了"——三种拒绝颗粒度都要支持。

**3. 商业目标只能搭便车**

商业目标（提升分期率、推广新产品、增加活跃度）只有在不损害用户体验的前提下才能嵌入。如果一次交互同时服务用户和商业，可以做；如果只服务商业，不做。

**4. 冲突时的决策透明**

当 Agent 做出"看起来不最优"的建议时（比如推荐利息更高的分期方案），需要有能力解释为什么。透明不等于啰嗦，一句话点明利弊就够。

**5. 原则的可审计性**

每一次交互都应该能被事后审查：这次建议是否符合三序原则？如果不符合，是在哪一步违反了？这要求交互日志中保留决策依据。

## 反模式

- **软推销**：用户拒绝分期后，Agent 在下次对话中用"你上次看的那个方案有更新"变相重推。这是对"用户自主排第一"的直接违反。
- **虚假关心**：以关心的口吻包装商业目标。"你最近消费有点多哦，要不要开个分期减轻压力？"——如果核心动机是拉分期率，这就是虚假关心。
- **选择架构操纵**：把商业目标对应的选项放在最显眼位置、用更大按钮、用引导性文案。Dark pattern 在情感陪伴场景下伤害更大，因为用户信任 Agent。
- **默认开启**：任何涉及商业利益的功能不应默认开启。默认 = 用户没有真正做出选择。

## 在花小呗中的实例

在花小呗的智能对话页中：
- 用户选择建议卡片后，Agent 回复始终以用户需求为中心（"我帮你看看有没有临时额度可以申请"），而非直接推产品
- 拒额五幕剧的第四幕「行动」中提供三个选项（备用金/分期/其他支付），均为平等呈现，无引导性排序
- 72 小时后回访是"关心结果"而非"再推一次"

## 适用边界

- 适用于：任何 Agent 有商业变现压力的场景（消费金融、电商、保险、教育）
- 不适用于：纯公益性 Agent（无商业目标冲突）
- 特别适用于：用户处于弱势地位的场景（借贷、保险理赔、医疗决策）——这些场景下违反三序原则的伤害更大
