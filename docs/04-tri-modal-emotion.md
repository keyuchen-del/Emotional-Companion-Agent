# 三模态情绪表达

## 一句话概述

纯文字 AI 的情绪表达天花板很低——三模态（动作 + 表情 + 文字）让情感陪伴 Agent 拥有文字 AI 无法复制的差异化护城河。

## 设计原则

**1. 三个模态各有分工**

- **动作**（视觉载体的姿态变化）：传递 Agent 的整体状态——开心时蹦跳、难过时低头、等待时踱步。动作是"远距离"感知，用户扫一眼就能 get。
- **表情**（视觉载体的面部/细节变化）：传递对当前话题的即时反应——惊讶、担忧、偷笑。表情是"近距离"感知，需要用户注意到细节。
- **文字**（对话文案）：传递具体信息和情感态度。文字是"阅读距离"感知，承载最多信息量但情感传递效率最低。

**2. 模态之间要一致，不要矛盾**

文字说"没关系"但表情在哭——这在人际交往中是微妙的信号，但在 AI 交互中只会让用户困惑。三个模态的情绪方向必须一致，除非你在刻意设计幽默效果。

**3. 视觉载体的选择是品牌决策，不是方法论**

Q 版吉祥物、3D 角色、emoji 组合、抽象动画——这些都是"视觉载体"的形态选择，取决于品牌调性和用户群体。方法论只规定"需要视觉载体来承载动作和表情"，不规定载体长什么样。

**4. 情绪变化有节奏，不要跳变**

从"开心"到"担忧"不应该是瞬间切换，而应该有过渡——先是开心减弱（动作放缓），然后表情变化（笑容收起），最后文字跟进（"等等，我看到一个情况..."）。节奏让情绪变化可信。

**5. 默认情绪是「温和在场」，不是「兴奋等待」**

Agent 不在交互时的默认状态不应该是"热情欢迎"，而应该是"安静待命"——像一个在旁边安静看书的朋友，你说话了 TA 才抬头。过度热情的默认状态会让用户有被监控感。

## 反模式

- **表情包轰炸**：每句话都配一个夸张的表情变化。情绪表达的频率应该低于对话频率——不是每轮都有情绪，有时候就是平静地在聊天。
- **emoji 替代真表达**：用 emoji 拼凑的"表情系统"。emoji 是通用符号，无法承载品牌性格和角色感。
- **纯装饰性动画**：视觉载体有动画但动画与对话内容无关。这不是情绪表达，是视觉噪音。
- **情绪过度化**：用户提到"今天有点累"，Agent 立刻进入"极度关怀模式"——过度反应和无反应一样令人不适。

## 在花小呗中的实例

花小呗采用 Q 版花朵吉祥物（🌸）作为视觉载体：
- **动作**：大号吉祥物在对话区底部，随对话状态变化（等待时有脚部小动画）
- **表情**：通过渐变色（#FFF0F2 → #FFBCC8）和阴影（rgba(255,180,190,0.25)）表达温暖基调
- **文字**：对话气泡中的文案风格一致——口语化、有语气词、用 emoji 点缀但不过量

## 适用边界

- 适用于：有视觉界面的情感陪伴 Agent（App、小程序、Web）
- 不适用于：纯语音 Agent（没有视觉通道，需要用声音的音调、节奏、停顿来替代动作和表情）
- 不适用于：API / 后台 Agent（无用户界面）
- 需要适配：多端场景下视觉载体的降级策略（手表端可能只有表情，无动作）
