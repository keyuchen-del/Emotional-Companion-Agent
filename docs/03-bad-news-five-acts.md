# 坏消息五幕剧

## 一句话概述

用户收到坏消息（拒绝、亏损、超支、降级）的瞬间，是情感陪伴 Agent 最容易失分也最容易得分的时刻——五幕剧将行业最冷漠的场景重构为有温度的交互链路。

## 设计原则

**1. 第一幕 · 通知：不报警，先陪你深呼吸**

坏消息的传达方式决定了用户后续 80% 的情绪走向。不要用系统通知的口吻（"您的申请已被拒绝"），而是用陪伴者的口吻（"这次没能帮你争取到..."）。关键：承认结果，但立即传递"我在你身边"的信号。

**2. 第二幕 · 共情：承认失望是合理的**

不要跳过情绪直接进入解决方案。用户需要被看见——"花了这么多时间准备，被卡在这一步，换我我也会难受。" 共情不是同情，是平等的理解。

**3. 第三幕 · 解释：透明但不技术**

解释为什么会这样，但用人话而非系统话。"最近 X 个因素让系统觉得现在不合适"比"您的综合评分未达到阈值"好一万倍。关键原则：**这不是你不够好，是时机不对。**

**4. 第四幕 · 行动：给真实选项，不给虚假希望**

提供 2-3 个具体可操作的替代方案，平等呈现。每个选项包含：是什么、代价是什么、适合什么情况。不引导、不排序、不隐藏。

**5. 第五幕 · 闭环：留一个温度的尾声**

结束时不是"感谢使用"，而是留下一个开放式的人情连接。然后——**72 小时后主动回访**。这是整个设计最关键的一笔。回访不是追销，是"那天的事你还好吗？" 这一步将 Agent 从"处理问题的工具"升格为"记得你的伙伴"。

## 反模式

- **冷通知**：用系统 notification 风格传达坏消息。"您的提额申请未通过。"——零共情，零出路。
- **跳过共情**：通知后直接给解决方案。用户感受：被当成待处理的 ticket。
- **虚假安慰**："别担心，下次一定可以！"——你不知道下次会不会可以，不要做承诺不住的安慰。
- **回访变追销**：72 小时后回访如果是"要不要再试试申请？"，那整个五幕剧的信任积累瞬间归零。
- **一次性交互**：坏消息处理完就结束，再也不提。用户感受：我只是一个 case number。

## 在花小呗中的实例

在花小呗 Demo 的「拒额五幕剧」页面中，以"小 A 在大麦买演唱会门票、花呗额度不够、临时提额被拒"为场景，完整呈现了五幕交互：

1. **通知**：「呜...小A，这次我没帮你争取到临时提额...」
2. **共情**：「花了这么多时间准备，被卡在这一步，换我我也会『诶』一下。」
3. **解释**：「真实原因我也告诉你：最近 X 个因素让系统觉得现在不能加。这不是你不够好，是时机不对。」
4. **行动**：三个选项平等呈现——备用金取用（利息更低）/ 分期付款（首期 X 元）/ 其他支付方式（家人代付或银行卡）
5. **闭环**：「搞定！演唱会结束后某天，我想听你说现场怎么样——你要记得回来告诉我哦」+ ⏰ 72小时后回访

## 适用边界

- 适用于：任何涉及"用户期望落空"的场景——贷款拒批、理财亏损、保险拒赔、服务降级、账户异常
- 不适用于：好消息传达（虽然好消息也需要设计，但不需要五幕式情绪缓冲）
- 慎用于：法律合规要求"必须用特定措辞"的场景——五幕剧的语气需要与合规要求兼容
