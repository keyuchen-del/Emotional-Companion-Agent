# 小满 Mun｜面向年轻人的消费陪伴 Agent

> 中性品牌实例 —— 把「情感陪伴」落到年轻人的日常消费场景

## 实例背景

- **产品**：独立中性品牌「小满 Mun」，一个陪年轻人过日子、做消费决策的 AI 伙伴
- **定位差异**：与首个案例 [花小呗](../huaxiaobei/) 不同，小满**淡化金融借贷**，聚焦「买什么 / 值不值 / 怎么省 / 钱花完后的情绪」。品牌哲学取自节气「小满」——**小满即圆满，刚刚好的拥有**，主动反过度消费，而非鼓励多花。
- **设计动机**：年轻人不缺购物工具，缺的是一个**不审判、有边界、会陪你一起判断**的伙伴。核心命题：把冰冷的消费数据，转化为一段温暖而克制的关系。

## 设计映射表

本 case 用到了通用手册 [docs/](../../docs/) 的全部 6 章：

| docs/ 章节 | 在小满 Mun 中的实例化 |
|---|---|
| [01 三层记忆架构](../../docs/01-three-layer-memory.md) | 先验层 = 平台已知的消费偏好（21 条）；共建层 = 用户亲口告知的身份/兴趣/目标/仪式（8 条）；默契层 = 长期互动沉淀（6 条）。「满记」页完整可视化 |
| [02 三序取舍原则](../../docs/02-three-order-principle.md) | 「值不值得买」由用户判断，AI 只给信息（页脚明示「陪你，但不替你做决定」）；五幕第四幕三选项平等呈现、无引导；72h 回访是关心而非追销 |
| [03 坏消息五幕剧](../../docs/03-bad-news-five-acts.md) | 以「限量联名球鞋发售、蹲了一周、手慢 30 秒没抢到」为场景，完整实现五幕交互 + 72h 回访 |
| [04 三模态情绪表达](../../docs/04-tri-modal-emotion.md) | 视觉载体 = CSS 手绘吉祥物「小满」（奶油团子 + 新芽）；动作 = 眨眼 / 拍拍挤压 / 漂浮 +1；表情 = 随五幕剧在 sad→think→joy 间切换。三者协同，是纯文字 AI 无法复制的护城河 |
| [05 关系成长体系](../../docs/05-relationship-growth.md) | 外化为「满值 62」+ Lv 体系 + 陪伴权益（全网比价 / 心愿单守护 / 月度复盘 / 专属表情包）+ 成长里程时间线 |
| [06 禁区锚点 + 未完待续锚点](../../docs/06-memory-anchors.md) | 禁区 = 粉色标签「别催我买东西 / 不提前任」；未完待续 = 蓝色标签「想入手一台相机，还没定型号」，附「续聊」入口。两者与普通记忆平级但视觉区分 |

## 小满特有设计（不进通用层）

详见 [design-notes.md](design-notes.md)。要点：

- 吉祥物「小满」形态（奶油团子 + 新芽 + 小脚，纯 CSS 绘制以便动画）
- 「小满即圆满」的健康消费观品牌定位与话术基调
- 暖色生活感配色（蜂蜜橙 `#FF9F45` / 新芽绿 `#5FBE63` / 莓粉 `#FF6F91` / 奶油底）
- 消费场景功能入口（值不值得买 / 降价提醒 / 月度复盘 / 心愿单）
- 球鞋发售场景的具体话术

## 技术说明

- 前端 Demo 为纯 HTML/CSS/JS 单文件，无外部依赖（仅引用 Google Fonts：Baloo 2 / Fredoka）
- 吉祥物、满值环、表情切换、漂浮反馈全部 CSS/JS 实现，无图片资源
- 适配移动端 400px 视口，建议用手机浏览器或 Chrome DevTools 手机模式查看
