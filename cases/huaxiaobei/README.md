# 花小呗｜首个情感陪伴 Agent 实例

> 支付宝 · 花呗 AI 助手的情感化设计落地

## 实例背景

- **产品**：支付宝花呗 AI 助手「花小呗」
- **设计动机**：花呗拥有海量用户数据和高频交互场景，但 AI 助手始终停留在"工具"定位——查额度、查账单、问客服。核心命题是：**如何把花呗生态里的冰冷数据，转化为一段温暖而有边界的 AI 关系。**

## 设计映射表

本 case 用到了通用手册 [docs/](../../docs/) 的所有 6 章：

| docs/ 章节 | 在花小呗中的实例化 |
|---|---|
| [01 三层记忆架构](../../docs/01-three-layer-memory.md) | 先验层 = 花呗生态数据（额度、账单、消费分，23 条）；共建层 = 用户主动告知的身份/兴趣/目标/仪式（7 条）；默契层 = 长期互动沉淀（8 条）。「小本本」页面完整可视化 |
| [02 三序取舍原则](../../docs/02-three-order-principle.md) | 建议卡片回复以用户需求为中心；拒额第四幕三个选项平等呈现无引导；72h 回访是关心而非追销 |
| [03 坏消息五幕剧](../../docs/03-bad-news-five-acts.md) | 以「大麦买演唱会门票、花呗额度不够、临时提额被拒」为场景，完整实现五幕交互 + 72h 回访 |
| [04 三模态情绪表达](../../docs/04-tri-modal-emotion.md) | 视觉载体 = Q 版花朵吉祥物（🌸），渐变色暖色调表达温暖；动作 = 脚部小动画；文字 = 口语化 + 适量 emoji |
| [05 关系成长体系](../../docs/05-relationship-growth.md) | 外化为「亲密度 V1」60 分 + 权益清单（闪购红包/闲鱼红包/分期免息/额度券/打车折扣）+ 赛季制 |
| [06 禁区锚点 + 未完待续锚点](../../docs/06-memory-anchors.md) | 禁区 = 红色标签「禁区 ⚠ 不要提前任」；未完待续 = 「上次说想聊聊毕业旅行计划」。两者与普通记忆平级展示但视觉区分 |

## 花呗特有设计（不进通用层）

以下设计决策与花呗品牌/产品深度耦合，不适合抽象为通用原则：

- Q 版「花小呗」吉祥物形态（🌸 花朵 + 绿色小脚）
- 拒额场景的具体话术（"呜...小A，这次我没帮你争取到临时提额..."）
- 支付宝 / 花呗 UI 设计语言对齐（配色 #FF4D4F、圆角风格、底部 Tab 结构）
- 花呗特有功能入口（查额度、快速涨额、消费人格速鉴、涨亲密度）
- 亲密度权益的具体内容（闲鱼红包、分期免息等花呗生态权益）
