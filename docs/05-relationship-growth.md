# 关系成长体系

## 一句话概述

情感陪伴 Agent 与用户的关系不应该是"恒温"的——需要一个可感知、可预期、有激励的成长机制，让用户愿意持续投入这段关系。

## 设计原则

**1. 成长是双向投入的结果，不是单方面打分**

不是"系统给你打了个分"，而是"我们一起走到了这个阶段"。成长指标应该反映双方的互动质量——Agent 的服务到位程度和用户的参与深度同时计入。

**2. 成长可感知、可预期**

用户应该能清晰地知道：我现在在什么阶段、下一个阶段是什么、怎么到达那里。模糊的"综合评估"不行——用户需要 mental model。

**3. 阶段跃迁有仪式感**

从一个阶段进入下一个阶段，应该有一个"时刻"——Agent 主动告知、表达欣喜、解锁新东西。这个时刻是关系积累的情感回报。

**4. 成长的外化是权益，但不只是权益**

权益（优惠券、免息券、专属服务）是成长的物质外化，但更重要的是情感外化——Agent 在高阶关系中展现更多"真实性格"、记住更多细节、给出更个性化的建议。用户感受到"TA 更懂我了"比拿到一张券更有价值。

**5. 关系可以退步，但不要惩罚性退步**

如果用户一段时间不活跃，关系可以自然冷却（Agent 的语气变得稍远一些），但不应该"降级惩罚"（"您的等级已下降，以下权益已失效"）。冷却是人情，降级是机器。

## 反模式

- **积分化**：把关系量化为积分，明码标价"多少积分换多少权益"。这把关系变成了交易。
- **日活绑架**："连续签到 30 天关系等级 +1"——这是留存指标伪装成关系设计。
- **单向评估**：只有系统在评估用户（信用分、活跃度、消费力），用户对 Agent 的服务质量没有反馈通道。
- **不可感知的变化**：用户感觉 Agent 从来没有变化，不管用了一天还是一年都是同一个反应模式。
- **赛季制焦虑**：如果采用赛季制（定期重置），赛季结束时的"归零感"需要被设计缓冲——否则用户感觉自己的投入被清空了。

## 在花小呗中的实例

花小呗将关系成长外化为「亲密度」体系：
- **可视化**：Header 区域显示当前亲密度分数（60 分）和等级（V1）
- **赛季制**：标注"本赛季 12.25-5.27"，有限时间窗口
- **权益清单**：按等级解锁——闪购红包、闲鱼红包、分期免息、专享额度券、打车折扣
- **涨跌机制**：底部提供"涨跌"入口，让用户了解什么行为会影响亲密度

## 适用边界

- 适用于：长周期陪伴型 Agent（用户预期使用数月以上）
- 不适用于：短周期工具型 Agent（用完即走，不需要关系积累）
- 慎用于：非商业场景的 Agent——如果没有权益可以发放，成长的外化需要完全依靠情感变化，设计难度更高
