# 情感陪伴 Agent 设计手册

> 6 章去品牌化设计原则，从真实产品落地中提炼

本手册适用于任何需要与用户建立长期情感关系的 AI Agent——消费金融、健康管理、学习辅导、生活助手等场景。每一章都从真实 case study 中提炼，而非空想。

## 章节导航

| 章节 | 主题 | 解决什么问题 |
|------|------|-------------|
| [01](01-three-layer-memory.md) | 三层记忆架构 | 如何让 Agent「开箱即懂你」而不产生「被监控感」 |
| [02](02-three-order-principle.md) | 三序取舍原则 | 用户意愿、系统建议、商业目标冲突时怎么排序 |
| [03](03-bad-news-five-acts.md) | 坏消息五幕剧 | 如何将最冷漠的拒绝场景重构为有温度的交互 |
| [04](04-tri-modal-emotion.md) | 三模态情绪表达 | 如何让 Agent 拥有文字 AI 无法复制的情感表达 |
| [05](05-relationship-growth.md) | 关系成长体系 | 如何让用户愿意持续投入与 Agent 的关系 |
| [06](06-memory-anchors.md) | 禁区锚点 + 未完待续锚点 | 市面 AI 尚无做好的两个记忆维度 |

## 每章结构

每章采用统一模板：

1. **一句话概述** — 这章解决什么问题
2. **设计原则** — 3-5 条可被任意 case 引用的原则
3. **反模式** — 特别警惕的 anti-pattern
4. **在花小呗中的实例** — 真实落地的参考
5. **适用边界** — 不适用什么场景
6. **延伸阅读** — 章节间的交叉引用
